# スライド設計書 MD 形式

スライドの内容とデザイン仕様を記述する設計書の書式。
Claude Desktop / Claude.ai などのツールに渡して PowerPoint を生成することを想定している。
デザイン定数（フォント・色・サイズ）をフロントマターに含めることで、
どのツールが読んでも同じ仕様で再現できるようにする。

## ファイルの置き場所

```
{分析ディレクトリ}/output/slides/slide_script.md
```

## 全体構造

```
[グローバルヘッダー]
---
[スライド1]
---
[スライド2]
---
...
```

## グローバルヘッダー（必須）

ファイル冒頭に YAML フロントマターで記述する。
デザイン定数はここに固定値として記載し、ツール側が参照できるようにする。

```yaml
---
title: 分析タイトル（短く）
subtitle: サブタイトルまたは補足情報
author: 担当者名
date: YYYY-MM-DD

# デザイン定数（変更しない）
slide_size: "10.0 × 5.625 inch (16:9)"
font_body: "Noto Sans JP"
font_code: "Consolas"
colors:
  navy:    "#0F1B3D"   # 見出し・ヘッダー背景
  blue:    "#065A82"   # 強調
  teal:    "#1C7293"   # セクション
  amber:   "#D97706"   # 警告（乱用禁止）
  ink:     "#1F2937"   # 本文テキスト
  muted:   "#64748B"   # キャプション・補足
  line:    "#E2E8F0"   # 区切り線
  bg_soft: "#F8FAFC"   # ソフト背景
  white:   "#FFFFFF"   # 反転テキスト・背景
---
```

## スライド1枚の書式

```
<!-- layout: content -->
<!-- notes: スピーカーノートをここに書く（1〜3文） -->

# スライドタイトル（主張を1行で）

- 箇条書き第1階層
  - 箇条書き第2階層（必要な場合のみ）
- 箇条書き第1階層

**図:** figures/figure01.png
**キャプション:** 図の内容を1行で説明
```

## layout 種別一覧

| layout 値 | 用途 | 必須要素 | 省略可能 |
|-----------|------|----------|----------|
| `title` | 表紙 | `#` タイトル, `##` サブタイトル | author, date |
| `section` | セクション区切り | `#` セクション名 | 説明文 |
| `content` | 本文（箇条書き主体） | `#` タイトル, 箇条書き | 図 |
| `figure` | 図主体（図を大きく見せる） | `#` タイトル, `**図:**` | 箇条書き |
| `figure-text` | 図＋箇条書き（左右分割） | `#` タイトル, `**図:**`, 箇条書き | — |
| `closing` | 結び・次のアクション | `#` タイトル, 箇条書き | — |

## 書式ルール

- `#` はスライドタイトル（1枚に1つ）
- `##` は表紙のサブタイトルのみに使う
- 箇条書きは `-` を使う（`*` 不可）
- 第2階層は必要最小限にとどめる（原則1階層）
- `**図:**` の後にファイルパスを書く（相対パスで分析ディレクトリ起点、例: `output/figures/fig01.png`）
- `**キャプション:**` は図がある場合に必須
- スピーカーノートは `<!-- notes: -->` に書く（Markdown ではなくプレーンテキスト）
- 1スライド = 1主張。タイトルだけ読んで結論がわかるようにする
